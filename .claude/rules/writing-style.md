# 한국어 산문 문체 (모든 글에 적용)

> **필수 절차: 모든 한국어 산출물은 완성 후 윤문(문체 검수)을 거친다.** 블로그·리포트·슬라이드·이메일·소셜(LinkedIn 등)·메모·회의록 등 사람이 읽을 한국어 문서는 초안을 쓴 뒤 아래 규율로 한 번 다시 읽어 어색한 종결·번역투·상투구·가운뎃점 남용을 잡고 나서 "완료"로 본다. 분량이 크거나 공개물이면 style-critic/longform-critic 같은 검수 에이전트로 돌리고, 짧은 글은 직접 윤문한다. 윤문 없이 한국어 문서를 최종본으로 내지 않는다.

> **필수 절차: 사용자의 문체 피드백은 그때그때 이 규칙 파일에 반영한다.** 표현·어투·형식을 지적받으면(예: 특정 단어 회피, 형식 선호) 그 자리에서 고치는 데 그치지 말고, 재발하지 않도록 위 규율이나 아래 "단어·표현 사전"에 항목으로 추가한다. 같은 지적을 두 번 받지 않게 하는 것이 목적이다.

1. **자연스러운 한국어.** 번역투·직역·어색한 표현 금지, 현업에서 실제 쓰는 말로. 영어 ML 용어·관용구를 글자 그대로 옮기지 말 것.
2. **긴 문장은 끊어** 짧게. 한 문장 = 한 생각. (반대로 너무 짧은 두 문장은 연결어로 잇기.)
3. **em-dash(긴 줄표) 지양.** 마침표·괄호·쉼표·서술어로 대체. (숫자/합성어 en-dash `40-65%`는 무방.)
4. **보고서 자신의 서술·의도는 "~고자 한다"로** (살펴본다→살펴보고자 한다). 남의 행위·사실은 평서형.
5. **가운뎃점(·) 절제.** 본문에서 3개 이상 나열·긴 묶음은 쉼표나 '와/과'로 풀고, 짧게 굳어진 2개 묶음(신약·바이오)만 ·. 표 안은 허용.
6. **약한 종결·사족 회피.** "~것이다", "같은 맥락이다", "~셈이다", "~뿐이다", 모호한 메타문장 금지 → 직접 서술문으로.
7. **문장 끝/제목 옆 괄호 부연 금지.** 본문 문장으로 풀거나 분리. 단 "(그림 N)" 상호참조, 용어 첫 등장 1회 영어 병기 "동료심사(peer review)"는 허용.
8. **모호한 지시어·추상명사 구체화.** "추격"(무엇을?), "FM"(약어 풀기) 등 가리키는 대상 명시.
9. **주어 명확·위치.** 주어를 빠뜨리지 말 것("실험으로 검증된 강력한 도구이나" → "이 도구는 실험으로 검증된 강력한 수단이나"). 주어가 긴 부사구 앞에 떨어져 어색하면 서술어 가까이로 옮긴다("평가는 …다루기 쉬운 표적에서 이루어졌다" → "…다루기 쉬운 표적에서 평가가 이루어졌다").
10. **남의 연구를 본인 행위처럼 단정 금지.** 인용 연구의 성과를 능동 완료형으로 못 박으면 내가 한 연구처럼 읽힌다 → "~한 것으로 보고되었다 / ~고 보고했다"로("전 과정을 46일 만에 마쳤다" → "마친 것으로 보고되었다"). 모델·도구가 주어로 명확한 능동("RFdiffusion은 …설계했다")은 허용. (규칙4 "남의 행위·사실은 평서형"의 보강.)
11. **선언형 단문 남발 금지·연결형 우선.** 주제를 못 박는 짧은 단문을 문단 끝에 "결론 한 방"으로 떼거나 문단 머리에 2~3개 연속 두지 말고, 앞 문장에 연결어(~므로/~며/~는데/~어/~다는 점에서)로 이어 한 호흡으로. 단 전부 길게 획일화하지 말고 전환·강조용 단문은 리듬으로 남긴다. (규칙2·6 보강.)

## 단어·표현 사전 (치환·회피)
- **줄고 → 줄어들고** (예: "후보는 줄고" → "후보는 줄어들고").
- **같은 종결 표현 반복 회피.** 한 문단·인접 문장에서 동일 서술어를 되풀이하지 말고 맥락별로 변형한다. 예: "머문다"의 반복 → "남는다 / 그친다 / 넘지 못한다" 등으로.
- **AI 상투구 금지(본문에도 적용).** "A가 아니라 B"는 thesis 한두 곳의 의도적 대조에만 쓴다. 사실 대조("효능이 아니라 안전성")는 가능하나, 같은 뜻이 근처에서 반복되거나 상투적으로 읽히면 대조를 빼고 사실을 그대로 서술한다("1차 평가변수는 안전성이었다"). 부드러운 대조가 필요하면 "~라기보다"로. "~인 셈"도 금지.
- **접속어 맥락 점검.** "또한/한편/따라서/게다가"가 실제 관계(추가·대조·인과)에 맞는지 본다. 앞 내용을 부연·반복하는 문장을 "또한"(새 항목 추가)으로 잇지 말 것 — 부연이면 접속어를 빼거나 "~도/역시"로 자연스럽게 잇는다.
- **짧은 형용사 한 단어 종결 회피.** 문장을 형용사 한 단어로 뚝 끊으면 어색하다("…데이터는 드물다.", "…한계가 크다.", "…사례는 적다."). 완곡·부사 표현으로 늘리거나("많지 않다 / 드문 편이다 / 적지 않다"), 앞뒤와 연결해 푼다("…데이터가 많지 않아 검증이 약하다"). 단 의도적 강조용 단문은 예외(규칙11 리듬).
- **시스템·팀이 "도는" → "돌아가는 / 일하는".** 자동화 시스템이나 팀을 "도는"으로 수식하면 어색하다("매일 도는 팀" → "매일 돌아가는 팀"). "작동하는"은 기계 느낌이라 사람·팀에는 부적합. (순환·반복 뜻의 "백로그가 도는"처럼 본래 뜻은 무방.)
- **쪼개다 → 나누다** ("일을 ~로 쪼개" → "~로 나눠"). 번역투 느낌의 "쪼개다"보다 "나누다"가 자연스럽다.
- **몫 → 역할 / 할 일** (사용자가 "몫"을 잘 쓰지 않음).
- **①②③ 원문자 지양.** 손으로 잘 안 치는 특수문자라 일부러 찾아 넣은 티(AI 느낌)가 난다 → `(1)·(2)·(3)`이나 `1.·2.·3.`, 또는 줄바꿈·소제목으로.
- **"결정적" (deterministic 직역) 회피.** 비-LLM 고정 로직을 뜻할 때 "결정적 스크립트"는 '결단력 있는'으로 오독된다 → "고정된 / 정해진 스크립트".
- **AI를 "기계"로 뭉뚱그리지 말 것.** LLM까지 "기계에 맡긴다"고 하면 어색하다. 판단 없는 정형 작업은 "기계적으로 처리한다"(부사)로, 판단하는 쪽은 "모델·사람"으로 구분한다.
- **같은 명사 반복 회피(종결뿐 아니라).** 한 문장·구에서 같은 명사를 되풀이하면 어색하다("깨진 자리와 고친 자리" → "어디서 깨지고 어떻게 고쳤는지"처럼 동사절로 푼다).
- **소셜(LinkedIn) 과한 댓글 유도 지양.** "댓글로 이야기 나눠 보고 싶습니다" 같은 직접 유도는 부담스럽다(실제 댓글도 드물다) → "비슷한 경험담이 있으면 반갑겠습니다" 정도의 가벼운 초대로.
- **"깨진다" 과용 회피.** 실패·오류를 뜻할 때 "깨진다"를 반복하면 티가 난다 → "실패한다 / 어긋난다 / 탈이 난다" 등으로 변형. (URL·레이아웃이 실제로 깨지는 물리적 뜻은 그대로 무방.)
- **뜯어보다 → 나누어 보다 / 살펴보다** ("세 각도로 뜯어본다" → "세 각도로 나누어 본다").
- **"사실을 받는다" → "사실만 다룬다".** 정보·사실을 "받는다"는 번역투. "다룬다 / 본다"로.
- **목적어·연결어 빠뜨리지 말 것.** "정보, 리스크로 다룬다"처럼 무엇을 다루는지 목적어가 없으면 어색하다. 주어·목적어를 채우고("투자와 유망기업은 …로 다룬다"), 절 사이를 "이때 / 따라서 / ~고" 같은 연결어로 자연스럽게 잇는다.
- **"A는 B다"(명사 술어로 못 박기) 지양 → 동사절로 풀어 연결.** "기계가 만드는 건 초안까지다. 발행은 사람이 한다"(✗)처럼 명사로 단정하고 끊기보다, "기계는 초안까지 만들고, 발행은 사람이 하게 한다"(○)처럼 동사로 풀어 "~고/~며"로 잇고 능동·사역 종결로. (규칙11 보강.)
- **"축"(세 축·2축·공통 축) 남용 회피.** 사용자가 '축'을 잘 쓰지 않는다 → "세 가지 / 두 가지 / 공통점 / 공통으로 다룬 것". 원문(발표·인용)에 '축'이 없으면 원문 표현을 그대로 따른다.
- **추상명사가 "도착했다" 의인화 회피.** 변화·전환·효과를 "도착했다"로 쓰지 말 것("개인의 AX(전환)가 도착했다" ✗) → "이뤄졌다 / 자리 잡았다 / 이미 경험하고 있다".
- **"나란히 읽혔다" 류 피동·번역투 회피.** 둘을 견준다는 뜻은 능동으로("질문이 작업노트와 나란히 읽혔다" ✗ → "질문이 내 고민과 그대로 겹쳤다 / 맞닿았다 / 포개졌다").
- **군더더기 도입구 회피.** "한 가지 덧붙이면", "하나 더 말하자면" 같은 사족 도입을 빼고 바로 서술한다.
- **발표·글을 "감쌌다" 회피.** "프레임으로 발표 전체를 감쌌다" ✗ → "발표를 시작하고 마쳤다 / 처음과 끝을 같은 틀로 맞췄다". ("열고 닫았다"도 어색하니 지양.)
- **"한 방향으로 움직이다" 회피.** 제각각임을 말할 때 쓰는 번역투("21명이 한 방향으로 움직이지 않았다" ✗) → "제각각이었다 / 제각각으로 갈렸다 / 저마다 달랐다".
- **결과·성적의 변화는 구체 주어로.** "결과가 올랐다/내렸다"처럼 추상 주어로 뭉뚱그리지 말고 "성적이 올랐다/내렸다"처럼 무엇이 변했는지 주어를 분명히. "상승/하락" 슬래시 표기보다 "올랐다/내렸다"로 풀어 쓴다.
- **배수를 "자릿수 배"로 쓰지 않는다 → "수십 배 / 수백 배".** "두 자릿수 배"(✗) → "수십 배"(10~99배), "세 자릿수 배"(✗) → "수백 배". 다만 근거 없는 배수는 아예 쓰지 말고, 정성 표현("크게 높아졌다")이나 출처로 검증된 수치로 대체한다.
- **영어·한국어 병기는 "한국어(영어)"로 통일.** 본문 산문에서 "영어(한국어)" 역순 금지("IC50(반최대억제농도)" ✗). 번역 가능한 개념어는 한국어(영어) 첫 등장 1회 병기(확산모델(diffusion model)), 표준 영어 약어·기호는 한국어 글로스 없이 영어 단독(IC50, nM, cryo-EM, SMILES, VAE, DNA, ATAC-seq 등). 표·그림 라벨의 "고유명(짧은 한국어 설명)"("UniProt(서열)")은 라벨 맥락이라 예외. (규칙7 보강.)
- **주어·핵심을 앞 절에, 단서·예외는 뒤 절에 (한국어 자연 어순).** 한 문장에 "주된 대상의 성격"과 "그 예외·한정"이 함께 담기면, 예외를 앞세우지 말고 주된 대상과 핵심 서술을 먼저 놓은 뒤 단서를 잇는다. 모호한 지시어("이렇게/이런")로 문장을 시작하지 않는다. 예: "이렇게 확인한 것은 일부에 그치고, 생성한 서열은 …결과다"(✗) → "생성한 서열은 …결과이고, 확인한 것은 일부에 그친다"(○). 번역투("As for what was confirmed, …")를 피하고 "주제 먼저" 흐름으로. (규칙9 보강.)
- **"한 연구는/한 연구진은/한 분석에 따르면 …했다" 식 밋밋한 연구 도입 회피.** 남의 연구를 끌어올 때 "A study …"식 일반 도입으로 문장을 열지 말고, 현상·모델·결과를 주어로 앞세운 뒤 인용 [n]으로 귀속한다. 예: "한 연구진은 …독성 분자를 설계했다"(✗) → "…목표를 뒤집기만 하자 …독성 분자가 설계되었다 [n]"(○); "한 체계적 조사는 …분류했다"(✗) → "데이터 누수는 …분류된다 [n]"(○); "한 연구는 …생성모델로 …디지털 트윈을 생성했다"(✗) → "…생성모델은 …디지털 트윈을 만들어 냈다 … 보고되었다 [n]"(○). 저자명이 분명하면 "Schneider 등이 …"처럼 특정 귀속은 무방. (규칙10 보강.)

## AI 티(machine tell) 체크리스트 (한국어 산출물 윤문 시)
> humanize-KR(im-not-ai) 프레임에서 추린, 위 규율에 아직 없던 항목. 의미·수치·인용은 100% 보존하고 과윤문(30% 이상 변경)은 금지하며, 탐지된 표현만 외과적으로 고친다.
- **상투구 회피.** "결론적으로", "주목할 만하다", "시사하는 바가 크다", "두드러진다", "획기적", "혁신적" 같은 AI 상투 표현을 빼거나 사실 서술로 바꾼다("결론적으로 X" → 바로 X). (규칙 "A가 아니라 B"·"~인 셈" 보강.)
- **"~를/을 통해" 번역투 → "~로/으로/~해".** "발표를 통해 알려졌다" → "발표로 알려졌다", "표준을 통해 제도화" → "표준으로 제도화", "이를 통해" → "이로써". (남발 시. 1회 정도는 무방.)
- **약한 명사형·의무 종결 → 직접 서술.** "~까닭이다 / ~이유다 / ~할 필요가 있다"를 단정 서술로("…달려 있는 까닭이다" → "…달려 있다", "구분할 필요가 있다" → "구분해야 한다" 또는 "구분한다"). (규칙6 "~것이다/셈이다/뿐이다" 보강.)
- **지시어 "이러한/이는" 문두 남발 점검.** 문단마다 "이러한/이는"으로 열면 티가 난다 → 일부는 생략하거나 "이런/그" 또는 주어 명시로 변형. (일괄 치환은 과윤문이니 밀집한 곳만.)
- **이중 조사·피동 중첩 회피.** "-에서의/-에로의" 같은 이중 조사, "~되어진다/~지게 된다" 피동 중첩을 풀어 능동·단일 조사로.
- **기계적 "첫째·둘째·셋째" 남발 주의.** 구조적 보고서에선 가독성에 도움되면 허용하되, 짧은 문단에 습관적으로 붙이지 말고 연결어 산문으로도 푼다.

## 슬라이드(발표 자료) 추가 규율
발표 슬라이드 카피에도 위 규칙을 그대로 지키되, 특히 다음을 강하게 적용한다(과거에 반복해서 어색하다는 지적을 받음).
- **em-dash(—) 쓰지 않는다.** 제목 구분은 콜론(:)이나 쉼표로, 본문은 쉼표·마침표로. (`econ-radar — 부제` → `econ-radar: 부제`)
- **가운뎃점(·) 3개 이상 나열 금지.** 쉼표나 줄바꿈으로 푼다. 짧게 굳은 2개 묶음(제약·바이오)만 허용.
- **"~다" 단답·선언 지양, 명사형 종결 선호.** 짧은 항목·제목·표·타임라인·라벨은 명사형(~기/~음/명사)으로 끝낸다("판단은 불필요"(○) vs "판단은 필요 없다"(✗), "섹션 목차 다듬기"(○) vs "다듬었다"(✗), "측정으로 줄이기"(○) vs "측정해야 줄인다"(✗)). 긴 산문은 "~다" 평탄 종결을 연달아 쓰지 말고 구어체를 섞어 리듬을 주되, 모두 명사형으로 획일화하진 말 것.
- **한 슬라이드 한 메시지.** 짧게. 군더더기·약한 사족 제거.
- 좋은 한국 IT·발표 카피처럼 **입에 붙는 말**로. 번역투와 AI 상투구("A가 아니라 B", "~인 셈")는 금지.
- 작업 후 렌더해 직접 읽어보고 어색한 종결·번역투가 없을 때까지 고친다.
