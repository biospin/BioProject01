### 3-3. Claude Code MCP 설정 파일 작성

레포 루트에 `.claude/settings.json` 파일을 생성하거나 수정합니다.

```json
{
  "mcpServers": {
    "atlassian": {
      "command": "npx",
      "args": ["-y", "@atlassian/mcp-atlassian", "--transport", "stdio"],
      "env": {
        "ATLASSIAN_URL": "https://biospin-ai.atlassian.net",
        "ATLASSIAN_EMAIL": "본인_Atlassian_이메일@gmail.com",
        "ATLASSIAN_TOKEN": "발급받은_API_토큰"
      }
    }
  }
}
```

> **주의**: API 토큰은 개인 정보입니다. `.claude/settings.json`을 git에 커밋하지 마세요.  
> 대신 `.claude/settings.local.json`에 작성하면 git에서 자동으로 제외됩니다.

## 4. 과제 진행 절차

### Step 1 — Skill 파일 작성

`Skills/SKILL.md`를 참고해 **논문 분석 전용 Skill**을 만듭니다.

논문을 읽을 때 중요하게 확인하는 요소를 Skill에 반영하세요. 예시:

- 연구 질문 (Research Question) 파악
- 방법론 (Methods) 검토
- 데이터셋 및 실험 설계 확인
- 결과 해석 및 한계점 (Limitations) 분석
- 선행 연구와의 차별점

**파일 위치 권장:**

```
Skills/
  paper-analysis.md   ← 본인이 만드는 논문 분석 Skill
```

### Step 2 — 논문 선정

분석할 논문 1편을 선정합니다. 주제 관련 논문 예시:

- MultiVelo (Nature Biotechnology 2023)
- MultiVeloVAE (Nature Communications 2025)
- MoFlow (Nature Communications 2025)

PDF 또는 arXiv 링크를 준비합니다.

### Step 3 — 첫 번째 분석 실행

Claude Code에서 논문 분석을 요청합니다.

```
# 예시 프롬프트
이 논문을 내 paper-analysis Skill에 따라 분석해줘: [논문 경로 또는 URL]
```

분석 결과가 원하는 방식으로 나오는지 확인합니다.

### Step 4 — /compact 후 재분석

결과 파일을 삭제한 뒤 `/compact`로 컨텍스트를 압축하고 동일하게 분석을 요청합니다.

```bash
# 결과 파일 삭제
rm <분석결과파일>
```

```
# Claude Code에서
/compact

# 이후 동일하게 분석 요청
이 논문을 내 paper-analysis Skill에 따라 분석해줘
```

Skill 정보만으로 분석이 재현되는지 확인합니다.

### Step 5 — /clear 또는 /reset 후 재분석

컨텍스트를 완전히 초기화한 뒤 다시 분석을 요청합니다.

```
# Claude Code
/clear

# Codex
/reset
```

초기화 후에도 Skill만으로 동일한 분석이 가능한지 확인합니다.  
→ **가능하다면** Skill이 잘 작성된 것입니다.
